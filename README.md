# Generative System Model (GSM)

The **Generative System Model (GSM)** is an open specification for systems that
**continuously define and govern their own state** — a vendor-neutral model of
governance, regulation, and adaptation grounded in systemics first principles.

GSM is a **Poesis** standard, published and stewarded under the Poesis name and
released as an open specification for the cloud-native community.

## The model in one paragraph

GSM is **definition-centric**: a system is described by eight primitives —
**Structure, Mechanism, Effector, Receptor, Interaction, Archetype, Directive,**
and **Norm** — governed by a **DNA grammar** (Directives, Norms, Ascriptions) and
an explicit **Ascription lifecycle**. Together they let a system express,
evaluate, and evolve the definition of its own state.

## Status

GSM is a **draft open standard** under active development. The model is
stabilizing; the specification text is versioned in this repository and rendered
on the docs site. A CNCF Sandbox application is in preparation — see
[CNCF-SANDBOX-APPLICATION.md](CNCF-SANDBOX-APPLICATION.md).
